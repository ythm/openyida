# 技能文档优化方法论（详细版）

> 本文档是 `skill-optimization-guide` 的参考文档，包含详细的操作步骤、分级标准、验证方法和反模式案例。

## 三层职责模型

```
SKILL.md（入口层）     → "怎么做" + "什么规则"（规则速查 + 完整开发流程）
references/*.md（规范层） → "为什么" + "代码细节"（详细规范 + 反模式）
samples/*.js（模板层）   → 可复制的起点（通过 openyida sample 获取）
```

### 各层职责边界

| 层级 | 内容 | 不应包含 |
|------|------|---------|
| SKILL.md | 规则摘要（名称+一句话）、开发流程（bash 命令）、API 速查表、参考文档导航表 | 完整代码示例、详细反模式解释、API 完整参数 |
| references/*.md | 规则的详细解释+代码示例+反模式、代码骨架（非完整实现） | 与 SKILL.md 重复的规则摘要、可运行的完整模板 |
| samples/*.js | 唯一的完整可运行模板，必须符合自身编码规范 | 规范说明文字 |

## 规则分级标准

将所有规则合并为统一的"核心规则"章节，分两级：

- **FATAL**：违反后代码无法运行（语法错误、运行时崩溃、API 调用失败）
- **IMPORTANT**：违反后代码能运行但有隐患（性能、体验、可维护性）

```markdown
### 致命规则（FATAL）
违反会导致页面崩溃或运行时报错：
1. **规则名称**：一句话描述

### 重要规则（IMPORTANT）
影响代码质量和用户体验：
1. **规则名称**：一句话描述
```

## 代码去重规范

| 文件类型 | 应保留的代码形式 |
|---------|----------------|
| SKILL.md | 零代码（只有 bash 命令和 API 速查表） |
| references/*.md | 骨架代码（函数签名 + 注释说明，不含完整实现） |
| samples/*.js | 唯一的完整可运行模板 |

### 去重操作

1. references 中的完整模板代码 → 精简为骨架，完整版指向 `openyida sample`
2. 多个 references 文件间的重复代码 → 保留在最相关的那个文件中，其他改为引用
3. samples/*.js 必须自检：模板代码是否符合自身编码规范

## 参考文档导航表规范

SKILL.md 末尾必须有完整的参考文档导航表：

```markdown
## 参考文档

| 文档 | 覆盖范围 | 何时阅读 |
|------|---------|---------|
| [编码指南](references/coding-guide.md) | 规范详解 | 编写代码前必读 |
| [宜搭 API](../../references/yida-api.md) | API 完整参数 | 调用 API 前必读 |
```

检查清单：
- 所有 references/ 下的文件都在表中
- 跨 skill 共享文档（yida-api.md、model-api.md）也在表中
- "何时阅读"列有明确指引（"必读" vs "按需"）
- 所有链接路径正确（特别注意相对路径层级）

## 模拟大模型阅读验证

优化完成后，让 AI 按以下顺序完整读取所有文档，输出评估报告：

1. 读取 SKILL.md（入口）
2. 按参考文档表顺序，逐个读取所有外链文档
3. 读取 samples/*.js 模板文件
4. 读取跨 skill 共享文档（yida-api.md 等）

### 评估维度

| 维度 | 检查项 |
|------|--------|
| 信息流 | 阅读顺序是否层层递进、无断裂？ |
| 冗余度 | 任意两文件重叠率是否 ≤ 15%？ |
| 一致性 | 规则数、规则内容、代码示例是否三者对齐？ |
| 自包含 | 仅读 SKILL.md 能否开始开发？ |
| 链接正确 | 所有相对路径是否可达？ |

### 问题分级

- **P0**：逻辑错误、规则遗漏、数字不匹配 → 必须修复
- **P1**：显著冗余、信息不一致、关键链接缺失 → 应该修复
- **P2**：措辞优化、格式统一、注释完善 → 可选修复

## 常见反模式

| 反模式 | 正确做法 | 实际案例 |
|--------|---------|---------|
| SKILL.md 内联完整代码 | 代码通过 `openyida sample` 获取 | SKILL.md 从 610 行降到 174 行 |
| 多个章节重复同一规则 | 合并为"核心规则"统一章节 | 4 个约束章节合并为 1 个 |
| references 包含完整模板 | 精简为骨架，完整版在 samples | coding-guide 87 行模板精简为 25 行 |
| 规则数声称与实际不符 | 交叉验证确保严格对齐 | "17 条"实际是 18 条 |
| 示例模板违反自身规范 | 模板必须自检合规 | template.js 有 3 个违规 bug |
| 流程指引单独成文件 | 内容少于 50 行直接合并到 SKILL.md | examples.md 43 行合并到快速开始 |
| 参考文档表缺少共享文档 | 补全所有相关文档（含跨 skill 共享） | 缺少 yida-api.md 和 model-api.md |
| 链接路径层级错误 | 仔细核对相对路径 | `./yida-api.md` 应为 `../../../references/yida-api.md` |

## 优化效果参考

以 yida-custom-page 为例：

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| SKILL.md 行数 | 610 | 174 |
| 文件数量 | 4 个文档 + 1 个模板 | 2 个文档 + 1 个模板 |
| 三文件代码重叠率 | ~70% | ~12% |
| 规则数对齐 | 不一致（17 vs 18） | 严格一致（18 = 18） |
| 模板合规性 | 3 个违规 bug | 0 违规 |
| 参考文档导航 | 缺少 2 个共享文档 | 完整覆盖 |
