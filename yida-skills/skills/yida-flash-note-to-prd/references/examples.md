# yida-flash-note-to-prd 使用示例

## 示例 1：将会议闪记转换为 PRD

### 输入

用户提供以下会议记录文本：

```
张三(产品经理)：今天主要讨论员工积分管理系统的需求。
李四（技术负责人）：好的，我们先说说积分的规则。
张三：积分分两种，一种是行为积分，比如打卡、参加培训；一种是贡献积分，比如提交代码、完成项目。
李四：积分有效期怎么算？
张三：行为积分有效期一年，贡献积分永久有效。
王五[HR]：积分可以兑换什么？
张三：可以兑换礼品、假期，后续还会加购物券。单次兑换上限 500 积分。
李四：需要审批吗？
张三：兑换需要 HR 审批，超过 200 积分的还需要部门主管审批。
王五：我们还需要一个积分排行榜，按部门统计。
张三：对，还需要积分明细查询，员工可以查自己的积分记录。
```

### 执行过程

```bash
# 方式 1：从文件读取
echo "上述内容" > meeting.txt
openyida flash-to-prd --file meeting.txt --name 员工积分管理系统

# 方式 2：直接发给 AI 智能体（推荐）
# 将文本直接粘贴到对话框，AI 自动识别并触发本技能
```

### 输出文件：`prd/员工积分管理系统.md`

```markdown
# 员工积分管理系统 — 产品需求文档

## 项目背景
公司需要建立员工积分管理体系，通过积分激励员工行为（打卡、培训、代码贡献等），并支持积分兑换礼品/假期等福利，提升员工积极性。

## 需求概述
建立员工积分管理系统，支持积分发放、查询、兑换及审批流程，并提供部门积分排行榜。

## 目标用户

| 角色 | 说明 |
|------|------|
| 普通员工 | 查看个人积分明细、发起积分兑换申请 |
| HR | 审批积分兑换申请、管理积分规则 |
| 部门主管 | 审批高额积分兑换（>200分） |

## 功能需求

### 功能模块 1：积分记录管理
- **功能描述**：记录员工的积分获取明细，支持按类型分类
- **业务规则**：
  - 行为积分：打卡、参加培训等，有效期 1 年
  - 贡献积分：提交代码、完成项目等，永久有效
- **字段设计**：
  | 字段名 | 类型 | 必填 | 说明 |
  |--------|------|------|------|
  | 员工 | EmployeeField | 是 | 获得积分的员工 |
  | 积分类型 | SelectField | 是 | 行为积分/贡献积分 |
  | 积分数量 | NumberField | 是 | 本次获得的积分数 |
  | 获取原因 | TextField | 是 | 如：参加培训、提交代码 |
  | 有效期至 | DateField | 否 | 行为积分填写，贡献积分留空 |
  | 备注 | TextareaField | 否 | 补充说明 |

### 功能模块 2：积分兑换申请
- **功能描述**：员工发起积分兑换申请，经 HR 及主管审批后生效
- **业务规则**：
  - 单次兑换上限 500 积分
  - 兑换需 HR 审批；超过 200 积分还需部门主管审批
  - 可兑换：礼品、假期（后续扩展购物券）
- **字段设计**：
  | 字段名 | 类型 | 必填 | 说明 |
  |--------|------|------|------|
  | 申请人 | EmployeeField | 是 | 发起兑换的员工 |
  | 兑换积分数 | NumberField | 是 | 最大 500 |
  | 兑换商品 | SelectField | 是 | 礼品/假期/购物券 |
  | 申请说明 | TextareaField | 否 | 补充说明 |

## 流程设计

**积分兑换审批流程**：
- 发起人：员工
- 节点 1：HR 审批（所有兑换申请）
- 节点 2：部门主管审批（仅当兑换积分 > 200 时触发）
- 通过后：积分自动扣减，兑换记录生效

## 数据报表

| 报表名称 | 数据来源 | 展示方式 | 说明 |
|---------|---------|---------|------|
| 部门积分排行榜 | 积分记录表 | 柱状图 + 排名列表 | 按部门统计总积分 |
| 个人积分明细 | 积分记录表 | 数据列表 | 员工查看自己的积分历史 |

## 待确认事项

| 序号 | 待确认内容 | 说明 |
|------|----------|------|
| 1 | 购物券兑换的上线时间 | 会议中提到"后续加购物券"，需确认排期 |
| 2 | 积分排行榜的更新频率 | 实时更新还是每日更新 |
```

## 示例 2：图片闪记识别

### 输入

用户上传一张会议白板照片（包含手写需求列表）。

### 执行过程

1. AI 使用视觉能力识别图片中的所有文字内容
2. 将识别结果作为闪记文本进入 PRD 生成流程
3. 输出结构化 PRD 文档

### 注意事项

- 图片识别依赖 Claude 视觉能力，确保图片清晰可读
- 识别结果可能需要用户确认关键信息后再生成 PRD
