# PRD 输出模板

> 本文档是 `yida-flash-note-to-prd` 技能的参考文档，定义生成的 PRD 文件的标准结构。
> 生成的高质量 prompt 遵循以下标准结构，写入 `prd/<项目名>.md`。

```markdown
# <项目名称> — 产品需求文档

## 项目背景
<!-- 从闪记中提取的项目背景、业务场景描述 -->

## 需求概述
<!-- 一句话描述核心需求 -->

## 目标用户
<!-- 从闪记中识别的目标用户角色 -->

## 功能需求

### 功能模块 1：<模块名>
- **功能描述**：...
- **业务规则**：...
- **字段设计**：
  | 字段名 | 类型 | 必填 | 说明 |
  |--------|------|------|------|
  | ...    | ...  | ...  | ...  |

### 功能模块 2：<模块名>
...

## 流程设计
<!-- 如果闪记中讨论了审批流程、业务流程等 -->

## 数据报表
<!-- 如果闪记中提到了统计、报表、数据分析需求 -->

## 非功能需求
<!-- 性能、权限、通知等非功能性要求 -->

## 待确认事项
<!-- 闪记中未明确或存在分歧的内容，需要用户进一步确认 -->
```
